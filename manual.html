1. Interface Layout

Kilmer's interface is divided into two areas: a left panel with all controls, and a canvas to the right where trees are built and edited.

Left panel sections

  • General: start a new tree (✦ New Tree) or open this manual (◫ Manual).
  • Files: load a previously saved file, save your current work, or export the diagram as an image.
  • Building direction: switch between Top-down and Bottom-up construction modes (see Section 2).
  • Tools: activate Move, Phase, Late Insertion, or Highlight mode.
  • History: undo or redo the last actions.

Canvas

The canvas displays the active tree. Hover over any node to reveal its action buttons. Zoom controls appear in the top-right corner of the canvas when a tree is present; the auto-save indicator appears in the top-left corner.

Active tools — Move, Phase, Late Insertion, and Highlight — are highlighted in green in the panel when enabled. Click the button again or press Esc to exit any active mode.

2. Building the Tree

Build modes

Select the build direction from the Building direction section of the left panel:

  • Top-down (↧): the buttons │, ⑃, and △ create daughters below the selected node. This is the default mode.
  • Bottom-up (↥): the buttons │, →⑃/⑃←, and △ create a parent above the selected node. Node action buttons appear above the node instead of below.

In Bottom-up mode, the arrows in →⑃ and ⑃← indicate where the sibling node will appear (left or right of the current node).

Node action buttons

Hover over any node to reveal its action buttons. The available actions depend on the current build mode:

  • │ — add a single daughter (Top-down) or a single parent (Bottom-up).
  • ⑃ — add two daughters via binary branching (Top-down), or wrap in a binary parent with a new sibling (Bottom-up).
  • △ — create a triangle for unanalyzed constituents.
  • +₂ — add a secondary label below the main node label (up to 4; see Section 3).
  • ✕ (red) — delete the node and all its descendants. On the root node, clears the entire tree while preserving the root label.

Double-clicking any branch line toggles it between a solid and a dotted style.

Terminal nodes and bottom labels

  • Use +₂ to add up to 4 secondary labels beneath a node's main label.
  • Each secondary label is editable (click to edit) and removable via its × button (visible on hover).
  • When a node reaches 2 or more bottom labels it becomes terminal (locked): existing daughters are removed and no new daughters can be added in Top-down mode. Bottom-up wrapping still works on locked nodes.

3. Editing Labels

  • Click any label to open an editable field directly over it.
  • Confirm with Enter or cancel with Esc to revert to the previous value.
  • While editing, click the Ω button in the floating node toolbar to open the special character palette. It includes IPA symbols, Greek letters, and mathematical symbols, and appears in the top-right corner of the canvas.

4. Text Formatting

Node labels support a lightweight markup system. Markers wrap the target text and are interpreted on render:

**text**Bold
*text*Italics
X_i or X_i_Subscript — renders as Xi
X¨n or X¨n¨Superscript — renders as Xn
#X#Overline
##X##Double overline
=X=Strikethrough

Combining styles

Markers can be nested to combine styles. For example:

  • **X**_i_ → Xi
  • *X*_i_ → Xi
  • X¨*n*¨ → Xn
  • #*XP*# → XP
  • =*t*_i_= → ti

5. Annotation Tools

All tools are accessible from the Tools section of the left panel. Activate a tool by clicking its button; it will highlight in green. Exit at any time with Esc.

Movement arrows (⤳ Move)

  1. Click ⤳ Move to enter Move mode. A status bar at the top of the canvas confirms you are in Step 1/2.
  2. Click the source node (e.g., a trace or a copy).
  3. Click the target node. A curved arrow is drawn beneath the tree connecting the two nodes.
  • Double-click an arrow line to toggle between dotted and continuous styles.
  • Click × on any arrow to remove it.
  • Arrows can represent movement, Agree relations, or other long-distance dependencies.

Phase boundaries (◠ Phase)

  1. Click ◠ Phase to enter Phase mode.
  2. Click a phase head node (e.g., a vP or CP node). A dashed arc is drawn above that subtree to mark the phase boundary.
  • Double-click the arc to toggle between dashed and solid styles.
  • Click × on the arc to remove it.

Late Insertion Model (⇔ Late Insertion)

  1. Click ⇔ Late Insertion to enter LIM mode. The cursor changes to a crosshair.
  2. Drag on the canvas to draw a dashed rectangle around the target region of the tree.
  3. A labeled node appears to the right of the rectangle, connected by a double-headed arrow. An input field opens immediately — type the phonological form or label.
  4. Use the Ω button next to the input to insert special characters.
  • Reposition the node by dragging it; the connecting arrow follows automatically.
  • Reposition the rectangle by dragging it independently.
  • Click × on the node, or press Delete, to remove the entire annotation (rectangle + node + arrow).

Oval highlights (⬭ Highlight)

  1. Click ⬭ Highlight to enter Highlight mode. The button color changes to the currently selected drawing color.
  2. To change color, click the ▾ chevron next to the Highlight button. Four colors are available: red, green, blue, and yellow.
  3. Drag on the canvas to draw an oval. A label input appears — type a label or leave it empty and press Enter.
  • Click an oval to select it and edit its label.
  • Drag a selected oval to move it; drag a handle to resize.
  • Click × on an oval to remove it.

6. Saving and Exporting

  • →◰ Save: downloads a .json file. This is Kilmer's native, fully editable format. The file can be reloaded at any time via ◰→ Load.
  • ↗ Export as image: generates a high-resolution PNG with clean rendering and appropriate padding, ready for inclusion in documents or presentations.
  • Auto-save: Kilmer periodically stores your work in the browser (indicated by the ☁ Saving… / ✓ Auto-saved chip in the top-left corner of the canvas). This is a safety net for accidental tab closes — it does not replace an explicit →◰ Save.
Tip: always download a .json file when you finish a session. The auto-save data is local to the browser and will be lost if you clear browser storage.

Zoom controls

The − and + buttons in the top-right corner of the canvas adjust the zoom level. Click the percentage indicator to reset to 100%. The zoom range is 40%–200% in 20% increments. Controls appear only when a tree is present.
